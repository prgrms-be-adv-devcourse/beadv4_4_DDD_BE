정산 배치 흐름 시각화

SettlementFlowTest의 내용을 기반으로 정산 배치의 전체 흐름을 확인할 수 있습니다.

1

1단계: 기본 멤버 데이터 초기화

SYSTEM, SELLER, BUYER 멤버 생성

2

2단계: 주문 생성 및 구매 확정

주문 생성 → 구매 확정 → 정산 후보 항목 생성

3

3단계: 일별 정산 수집 배치 실행

CandidateItem을 수집하여 Settlement(정산서) 생성

4

4단계: 테스트를 위해 정산서 기간 조정

정산서 기간을 저번달로 조정

5

5단계: 월별 정산 완료 배치 실행

저번달 정산서에 대해 지급 완료 처리

6

6단계: 최종 정산서 조회

해당 월의 최종 정산서 데이터 확인

단계 1 / 6

SYSTEM, SELLER, BUYER 멤버 생성 주문 생성 → 구매 확정 → 정산 후보 항목 생성 CandidateItem을 수집하여 Settlement(정산서) 생성 정산서 기간을 저번달로 조정 저번달 정산서에 대해 지급 완료 처리 해당 월의 최종 정산서 데이터 확인

  • SYSTEM 멤버 생성 (시스템 수수료 수집용)
  • 판매자(SELLER) 멤버 생성
  • 구매자(BUYER) 멤버 생성
  • 주문 상품 생성 (3개 상품: 10,000원 + 25,000원 + 5,500원 = 40,500원)
  • 주문 상태 변경: 결제 완료 → 배송 완료 → 구매 확정
  • 정산 후보 항목(SettlementCandidateItem) 생성
  • 총 판매 금액: 40,500원
  • 일별 배치 실행 (매일 03:00 자동 실행)
  • 정산 후보 항목을 수집하여 정산서 생성
  • 판매자 정산서: 판매대금 = 총액 - 수수료(10%) = 36,450원
  • 시스템 정산서: 수수료 = 총액 * 10% = 4,050원
  • 월별 배치는 저번달 정산서만 처리하므로 기간 조정 필요
  • 정산서의 settlementYear, settlementMonth를 저번달로 변경
  • 월별 배치 실행 (매월 25일 04:00 자동 실행)
  • 저번달 정산서에 대해 지급 완료 처리
  • payoutAt 필드에 지급 일시 기록
  • 년도와 월을 선택하여 정산서 조회
  • 정산서 요약 정보 확인 (총 판매 금액, 수수료, 정산 금액, 지급 일시)
  • 정산 항목 상세 정보 확인
